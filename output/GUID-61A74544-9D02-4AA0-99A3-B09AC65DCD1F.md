# Estimating a single OD matrix from a multiple class problem

It may be desirable to estimate only a single OD matrix from
a multiple user class
Avenue assignment. This could occur such as in a case
where the user may wish to use different input parameters for each class, or
may only be concerned with a single class at the moment. In this case, the
[DYNCLASS](GUID-876F348E-7006-40A3-A8CD-7711B465AE97.html)parameter can be used to define the class the user wishes to
estimate. When using
[DYNCLASS](GUID-876F348E-7006-40A3-A8CD-7711B465AE97.html),
you must provide the
Analyst Drive program with a single dynamic count
file, a single OD matrix for the desired class, and the
Avenue packet log file. The
[DYNCLASS](GUID-876F348E-7006-40A3-A8CD-7711B465AE97.html)
parameter should then be set to the corresponding class number as defined by
the
Avenue assignment.
