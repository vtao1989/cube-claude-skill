# CubeAPI Classes Summary

The below table provides a summary of the main classes that
the CubeAPI provides. This list is not fully comprehensive but is intended to
introduce the main components of the API and their functionalities.

| Utility Category | CubePy Class | Description |
| --- | --- | --- |
| CUBE Network / CUBE Database | BNFSConfig | This class allows to create a new CUBE SQLite network by importing a network from an ESRI shape file.  For node numbering, the user can use existing node numbers from the shape file or create new numbers.  If the existing numbers should be used and both the node and link files are provided, an attribute called N should store the node number and two attributes named A and B should be included in the links Shape File, with the correspondence to N. |
| CubeDatabase and derived objects | This class provides the capability of creating a new, opening an existing, and working with a CUBE database. Currently a CUBE database is a Spatialite or PostGIS database.
