# Estimation of Static PT Matrices

A new feature in
Analyst Drive allows users to estimate static PT
matrices in the same manner as static HIGHWAY matrices are estimated. To
utilize this feature, users simply include PT matrices, screenline files and
ICP files in place of HIGHWAY versions of these files. All features of static
estimation that are available for estimating HIGHWAY matrices will also work
with PT matrices (e.g. aggregate estimation with
[ICPARRAY](GUID-3C98976F-C934-4326-B74F-6BC80466E726.html)
and
[SLCODE](GUID-33A3AB6F-E1EB-4F2C-911C-6E70894ED1C2.html),
multiclass estimation with
[SLCODE](GUID-33A3AB6F-E1EB-4F2C-911C-6E70894ED1C2.html),
confidence matrices, etc.). The user can also include PT and HIGHWAY files for
a multiclass estimation in a single Analyst Drive run. The user should note
that a special format specifier for the ICP files is not required for PT
matrices as it is for the HIGHWAY case, and so the FORMAT identifier should not
be used in the PT script.
Analyst Drive does not yet allow the user to use PT
Lines files in the estimation process as is available with the original
Analyst program.
