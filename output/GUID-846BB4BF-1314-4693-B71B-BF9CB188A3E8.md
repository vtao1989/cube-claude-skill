# General route choice probability matrix files

If using the
[GENERAL](GUID-537E9D14-8742-4648-BCD5-93CC253CD77F.html)=T
option then the user must provide user generated route choice probability
matrix files which take the data (in order): Zone I, Zone J, Time D, Count
Number K, Time T and Probability value. There should be NO HEADER included in
the file; for 100 zones the route choice probability matrix file should follow
the format of Figure 14. These files should match corresponding turning count
files. The specifier that defines the general route choice probability matrix
in the general dynamic control file (corresponding to screenline counts) is
given by IRCM, and IRTM defines the route choice file for dynamic turning
counts.

Note: When running an estimation from an
Avenue
assignment using the option
[KEEPFILES](GUID-2AE231C3-8809-48D7-A3CF-D99C025E4CC5.html)=T, general
format route choice probability matrix files will be placed in the folder path
defined by the
[WORKDIR](GUID-DD46DFCC-6C37-45AC-842C-964556F6ED8F.html)
parameter and will have the names INPUT\_ROUTE\_CHOICE\_MATRIX\_#.DAT for
route-choice matrices corresponding to screenline counts, and
INPUT\_TURN\_ROUTE\_CHOICE\_MATRIX\_#.DAT for route-choice matrices corresponding to
turning counts
