# Dynamic Matrix Estimation with Partial Trip Data

A new feature available in
Analyst Drive Version 1.5.1 allows users to include
partial trip count data when performing dynamic matrix estimation. Partial trip
data is commonly available in many traffic networks with the use of advanced
modern detectors such as Bluetooth detectors and can also come from more
traditional sources such as license plate surveys. To use partial trip data in
the dynamic estimation process users are required to perform the dynamic
estimation using
Cube Avenue in order to produce a packet log file.
Analyst Drive produces partial trip route proportion
matrices by processing the packet log in a similar fashion to producing
traditional route choice and turn choice matrices.
