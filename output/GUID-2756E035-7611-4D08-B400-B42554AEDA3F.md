# Dynamic OD matrix files

The dynamic OD matrix file accepted by the default dynamic estimation
option to use in conjunction with
Cube Avenue is a
Cube matrix file with a matrix tab for each time
interval. Confidence matrices for dynamic estimation should follow as a group
of matrix tabs, and should contain a confidence matrix tab for each input time
segmented OD matrix tab. This group should follow in order after the last time
segmented OD matrix tab. Differing from the static matrix case, dynamic input
matrix files should only contain OD matrices for a single user class, but
should contain all time periods for a single class in sequential order in an
individual matrix. The
Analyst Drive module block accepts multiple dynamic
matrix files, and one should be provided for each user class used in the
Avenue assignment.
