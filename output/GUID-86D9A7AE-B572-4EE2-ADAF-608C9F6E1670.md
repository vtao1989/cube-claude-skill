# CUBE 7 Cluster Introduction

CUBE 7 Voyager Cluster is a significant restructure of the
CUBE Voyager Cluster functionality for controlling the parallel Voyager
functionality in the Matrix, Highway and Pilot modules.

The previous system of using "signal files" to pass
commands, status and results between cluster nodes has been replaced by a
networking messaging system, meaning the two systems are incompatible.

The use of direct network connections results in more
reliable passing of messages which are sent and received in order, unlike using
the file-system which optimizes and caches reads and writes which can lead to
inadvertent delays when files are accessed remotely from different machines.
