# Full Model Run

For this exercise, we will learn how to perform a full
model run. It is necessary to perform such a run the first time in order to
obtain data for every phase of the model, e.g. trip generation, distribution,
mode choice, assignment, etc.

1. In any project file, you can begin a full model run by
clicking on Toolbar > Run. This will open the Batch Run Composer (BRC).

![](GUID-30CEF835-4532-4AC7-9A91-C63A42A05FC5-low.png)

![](GUID-7911E253-5CA6-4871-91C9-C2B8B6295D3F-low.png)

- You can also access the BRC
  from the Project Explorer.

![](GUID-AF97CEEC-3753-419E-8986-2173587C18A6-low.png)

- Tip: If you
  have the Scenario Manager open for a particular scenario, clicking Run will
  open the BRC.

![](GUID-21ADCA8F-BEFE-403D-BC34-B86F47D022AF-low.png)

2. In the BRC, select an application from the list. The
application will appear in the Applications to Run box.

![](GUID-45868C32-8DA3-4A0E-9F39-DE538BEEC2B0-low.png)

3. Select a scenario from the list. The scenario will appear
in the Scenarios to Run box.

![](GUID-B08D0E9C-A48E-4FF7-91FB-8BABF0303594-low.png)

4. Click Run. For this exercise, the Cluster pools, Stop
Batch on Any Application Error, and Autosave Log Files During Runs to Logs
Folder will not be covered.

![](GUID-3407BF19-A7CA-4CAD-85A9-0914276B29C9-low.png)

5. You should be able to see both the Run Status Monitor and
Application Run Monitor while the model run is active. The former will provide
scenario name and code, which application is being ran, start and end time. It
will also provide the status of the run and any error/warning messages once
completed. The latter will allow you to trace where the run is currently
located and show run times for each completed element.

![](GUID-1773DBD7-3FFB-427D-8141-3521FB8A2D8C-low.png)

6. Click Close when the run completes. You can also click
on the error/warning code to see the message.

![](GUID-C2221783-09C5-4DD4-9128-7C738423266A-low.png)
