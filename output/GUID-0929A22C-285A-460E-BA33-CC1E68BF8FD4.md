Scenario Editor is the interface that allows to define,
edit, run scenarios, as well as review input and output data by scenario.

- To Edit Scenarioâs inputs:

1. Open the Scenario Editor:

- Double-click on the scenario
- Right-click on the
  Scenario > Open

![](GUID-016613FB-D0CF-415E-A984-522FB02E64DF-low.png)

2. Select from the green drop-down the keys you want to
display, either application specific keys or all the project keys.
  
![](GUID-CE5C5ADD-57E0-44F5-8C85-F4A21FE2F4D7-low.png)

3. Edit the key values as needed.

Note: If a Key value is
non-zero, you cannot edit, delete and keep the value as empty. The key will
still expect a non-zero value for the respective scenario.

4. To save new key values click on the Save Project file in
Project Explorer.
  
![](GUID-454D8F0A-6EEA-4EB7-A7C8-8A8A65CE7CE1-low.png)

- To revert to applying the parent scenario key value to a key:

Note: By default, new scenarios will inherit the key values of the parent
scenario

 > 
Inherit

Tip: When a
key is NOT Inheriting a value from its parent scenario the key's title is
highlighted:
  
![](GUID-8D4E874C-E178-4F36-8211-85B54BE143A7-low.png)

- To change all scenario specific key
  values to inherit the parentâs key value instead:

   and right-click on the title of the key.
2. Select
   Propagate Value to Descendants (this option is
   only enabled for parent scenarios)
     
   ![](GUID-29E35B33-7A3F-4164-A554-19535ECF0051-low.png)

- To retrieve the default value for a scenario specific key:

Right-click on the key title>
Apply default value.
  
![](GUID-1A2FAC21-C392-414F-8540-FA77A284B9C1-low.png)

- To run the Application and Scenario from the Scenario Editor.

1. Click on the run symbol. In the Batch Run Composer
select the appropriate Application(s) and
Scenario(s) > Run
