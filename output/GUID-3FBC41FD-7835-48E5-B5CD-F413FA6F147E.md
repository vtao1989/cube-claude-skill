# Performance metrics with NCORES and MPICORES

To help understand how parallel performance is affected by the use of
the
[NCORES](GUID-E23E89B8-1CBA-41D1-A643-788A7C2CC72B.html) and
[MPICORES](GUID-539DD601-CF95-495D-8993-D058970BF046.html)
parameters we will examine several cases for each and try to explain why
individual problems are able to achieve better parallel performance than
others, and how differences in using the two parameters can affect overall
speed. The goal of this section is to help the user better understand how to
use these parameters to achieve maximum performance benefit for their own
models.

The three cases that we will be studying vary in size and scope to give
a feel for how different problems benefit from both the
[MPICORES](GUID-539DD601-CF95-495D-8993-D058970BF046.html)
and
[NCORES](GUID-E23E89B8-1CBA-41D1-A643-788A7C2CC72B.html)parameters. All three cases use multiple user classes since we can
only make comparisons for the
[MPICORES](GUID-539DD601-CF95-495D-8993-D058970BF046.html)
parameter in such cases. While we won't go into specific details on the full
model of each case, the important data aspects for each are:

1) and 42 screenline counts (class 2).

3.

nine user classes, 10,000+ Screenline counts, and 6000+ zones

We will first examine using
[NCORES](GUID-E23E89B8-1CBA-41D1-A643-788A7C2CC72B.html)and
[MPICORES](GUID-539DD601-CF95-495D-8993-D058970BF046.html)
individually, and finally bring the two together to show how the maximum
parallel performance benefit can be achieved through a combination of the two.
