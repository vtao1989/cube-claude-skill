# Other Considerations

Public Files to
Scopes

CUBE 7 introduces a new way (scopes) to link files across application
groups. This replaces the public files implementation in CUBE 6. For more
information on how to use scopes, please refer to
[Organizing file
Linkages (Scopes)](GUID-8438A5F1-ACFF-4445-A758-82B8E30B7F45.html)

External programs

If there are external programs in your model process, it would be a good
check to the required folder structure and other input files are available. The
importer identifies input files for CUBE programs and copies them to the new
project location, but it cannot check for input files for external programs.

TRDEL/TRCOPY

The TRDEL and TRCOPY program boxes have been depreciated. These are used
to delete and copy files can be replaced with a simple PILOT statement as below

```
*del <filename>
```

```
*copy <input file> <output file>
```

User Programs

The customized user program boxes from CUBE 6 have been depreciated.
This can be replaced with a simple PILOT statement as below are using a CubePy
program box. For more information on how to set-up a CubePy program box, refer
to
[Adding CubePy
programs](GUID-8BD83231-32D9-4049-94AF-1674D40158D9.html)

```
*<Program executable> <arg1> <arg2>...
```
