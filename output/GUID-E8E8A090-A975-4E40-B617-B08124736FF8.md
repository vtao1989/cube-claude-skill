# Trip end files

If using the
FILETE=T option then the user must provide user
generated trip end files for each user class which take the data (in order)
Zone number, trip end I-total, trip end J-total, confidence I, confidence J.
There should be NO HEADER included in the file; for 100 zones each trip end
file should follow the format of Figure 11.

![](GUID-524CF690-D542-4DD8-8C62-D93F51BA3445-low.png)

Figure 11: Trip end file format. Each row should
contain in order (1) zone number, (2) trip end I-total, (3) trip end J-total,
(4) I-confidence, (5) J-confidence. The zone number (first column) must be
integer type, the remaing values can be integer or floating point. The file
should contain no header.
