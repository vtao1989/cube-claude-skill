# Estimating multiple matrices with as many data sets

When estimating multiple matrices, it is ideal to have an
individual count set and ICP file for each matrix. To estimate multiple
matrices when each matrix has an individual screenline and ICP file, the user
simply needs to add these to the input control box and set the
[NCLASSES](GUID-A93296A2-561D-4E9B-807E-3114ACB9B4D8.html)
parameter to the total number of matrices to be estimated, and no other
modifications need to be made. Using this method, the
Analyst Drive program will go through and estimate
each matrix individually in a sequential fashion. If using the
[MPICORES](GUID-539DD601-CF95-495D-8993-D058970BF046.html)
parameter with ANALYST2\_64, the value for
[MPICORES](GUID-539DD601-CF95-495D-8993-D058970BF046.html)
may be as high as the value for
[NCLASSES](GUID-A93296A2-561D-4E9B-807E-3114ACB9B4D8.html).
When using
[MPICORES](GUID-539DD601-CF95-495D-8993-D058970BF046.html),
Analyst Drive will simultaneously estimate the number of matrices defined by
the
[MPICORES](GUID-539DD601-CF95-495D-8993-D058970BF046.html)
parameter in a parallel loop until all matrices have been estimated.
