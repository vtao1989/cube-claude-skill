# Analyst Drive control file

The
Analyst Drive program requires a control
file which provides the program with the necessary input and output files to be
used along with a list of parameters and options. This control file is
automatically generated when using the Application Manager block diagram and
the graphical interface provided. To use the graphical interface, simply right
click on the control file name and choose the
Edit option. Alternatively, one can write the
control file themselves (or manually alter one that has already been created)
by right clicking the control file box and selecting
Text Edit. An example of the control file is given
in Figure 4.

When running
Analyst Drive from Application Manger,
catalog keys can be used in file names as in the example file above. However,
it may be important for the user to be able to run the
Analyst Drive program from the command
line (see
[ÂRunning from the command lineÂ](GUID-FF93B941-C764-4503-905C-4C18914808F0.html#GUID-FF93B941-C764-4503-905C-4C18914808F0)). In this case, the user must manually replace all catalog keys
in braces, e.g. {My\_Key}, with the correct characters or values since these are
replaced with the actual values when running from
Cube.
