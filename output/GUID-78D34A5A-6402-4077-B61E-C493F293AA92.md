# Estimating multiple matrices with fewer data sets

In many cases it may be desirable for the user to estimate
multiple matrices from fewer sets of count data. For static estimation, this
can be done by the
Analyst Drive program using one of two different
methods. The first method preprocesses the screenline count data and produces a
count file for each matrix to be estimated. Thus, the process can be carried
out as in the base situation where there are as many count sets as matrices to
be estimated. The second method uses aggregate totals and corresponds to the
optimization problem given by Equation Set 5. The full description of the two
methods and how to implement each are given here.
