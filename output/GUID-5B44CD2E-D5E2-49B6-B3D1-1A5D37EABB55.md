# Build Applications - Finding Items Inside Applications

In this tutorial, we will look at how to find items inside applications.
The Search Application function from the Toolbar is a new addition which allows
users to find the exact location of a group, program, file, and so forth
located within a model.

1. To begin, click on Toolbar > Tools drop-down arrow > Search.

![](GUID-44173AB0-0EEC-42CD-B3AB-42066B680278-low.png)

- You can also access this by
  right-clicking anywhere in the Application Editor > View > Search.

2. Type the name of an item associated with the application in the
search box to filter it.

![](GUID-16A160FF-EC8B-42D9-89EF-7B6A4FC24428-low.png)

- Tip: By using Exact Match, you can isolate the item to a
  single filter.
- Note: If the item is located in an item of a higher level (e.g. file
  that is a part of a program), you must type down the entire path if using the
  Exact Match option.

![](GUID-4D78E392-49EE-4051-8058-B6C6265CA38F-low.png)

3. There are three filters for class types: Select All Class Types,
Clear All Class Types, and Invert Selected Class Types. For the last option, an
example is if the four class types in the first row were selected and that
filter option was used, the first row would be de-selected and the second row
would be selected. Only checking one class type box will have CUBE search items
only in that class type.

![](GUID-9548D716-B436-4547-8EFD-927695937691-low.png)

4. After applying a filter > click Go to Selected.

- You can also double-click
  the item to transport to its location.
- Note: If you selected a file, CUBE goes to the element that is holding
  it (loop, program, etc.) and highlight the file.

![](GUID-E06D9D04-3155-4C3E-B9BB-092C3CA2D767-low.png)
