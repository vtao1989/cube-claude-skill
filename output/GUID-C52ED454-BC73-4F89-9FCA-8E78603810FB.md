# Estimation with more time intervals than count sets

In the case that the
Cube Avenue assignment contains more time intervals
than available periodic count sets, the user can aggregate the matrices they
wish to estimate over a period and perform the estimation on the aggregated
matrices using the
[AGGINTRV](GUID-9BC2654C-1D5B-4048-AB44-40F149568E10.html)
parameter. For example, if there were an 18-hour assignment period using
Cube Avenue, but one only possessed counts every three
hours, then the assignment could be run through
Avenue using the 18 one-hour periods. Then, the user
could aggregate the matrix into six three-hour period matrices. In this
scenario the
[AGGINTRV](GUID-9BC2654C-1D5B-4048-AB44-40F149568E10.html)
parameter must be set to the number of time periods that are being aggregated
so that
Analyst Drive can adjust the route proportion matrix
derived from the packet log file. In the above example, [AGGINTRV](GUID-9BC2654C-1D5B-4048-AB44-40F149568E10.html) should be set
to 3 because three time periods are being combined to form a single matrix for
the estimation.

Note: This only works for
periodic time intervals; i.e., the counts present must all come from a defined
period of hours, which in the example above is 3. It will not work if all
dynamic intervals which form the time aggregated dynamic matrix are not equal.
